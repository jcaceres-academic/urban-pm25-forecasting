🧩 Overview

This section documents the Python scripts implementing the full analytical pipeline for PM2.5 modelling in Madrid.

The repository is structured as a sequence of modular scripts. Each script corresponds to a specific stage of the workflow, from data preparation to hybrid forecasting.

The objective is to ensure methodological transparency and reproducibility, rather than interactive execution.

📂 Script organisation

The workflow is organised into the following stages:

  |

  |

  |
  ├── 06_station_map.py 
  |
  ├── 07_hybrid_forecasting.py 
  |
  ├── 08_district_forecasting.py

Each script can be executed independently, although they are designed to follow a logical sequence.

🧱 Data preparation

01_data_preprocessing.py

This script transforms raw air quality datasets into a structured format suitable for analysis.

It performs:

  • Temporal parsing and validation

  • Reshaping daily measurements

  • Filtering relevant pollutants

  • Integration with station metadata

  • Generation of clean datasets

The output consists of processed CSV files per year, which are reused across all subsequent stages.

🗺️ Spatial analysis

02_spatial_analysis.py

This script generates district-level representations of PM2.5 concentrations.

It includes:

  • Aggregation of data by district and year

  • Harmonisation of district naming conventions

  • Integration with geographic shapefiles

  • Production of choropleth maps

These outputs support the interpretation of spatial variability in air pollution.

📊 Descriptive analysis

03_descriptive_analysis.py

This stage provides a statistical overview of PM2.5 behaviour.

The analysis includes:

  • Summary statistics by year

  • Distributional analysis

  • Identification of variability patterns

These results help characterise the structure of the dataset prior to modelling.

📈 Temporal trend analysis

04_trend_analysis.py

This script examines temporal dynamics across districts.

It focuses on:

  • Aggregation of PM2.5 levels over time

  • Comparison between districts

  • Identification of heterogeneous patterns

This step provides context for understanding spatial and temporal dependencies.

📍 Monitoring network representation

05_district_map.py

06_station_map.py

These scripts generate spatial visualisations of the monitoring infrastructure.

They include:

  • District boundaries

  • Locations of monitoring stations

  • Identification of districts with PM2.5 measurements

These maps provide essential geographical context for the analysis.

🤖 Hybrid forecasting model

07_hybrid_forecasting.py

This script implements the core Prophet–LSTM hybrid model.

The modelling pipeline consists of:

  • Daily aggregation of PM2.5

  • Prophet-based trend and seasonality modelling

  • Residual computation

  • Normalisation of residuals

  • LSTM-based learning of nonlinear patterns

  • Forecasting of residual dynamics

  • Reconstruction of the hybrid prediction

The output corresponds to the main forecasting results presented in the study.

🏙️ District-level forecasting

08_district_forecasting.py

This script extends the hybrid modelling framework to individual districts.

It includes:

  • Filtering of district-specific data

  • Independent model training per district

  • Local forecasting

  • Comparison between predicted and observed values

This approach highlights spatial heterogeneity in predictive performance.

⚙️ Execution notes

The scripts were originally developed in a cloud-based environment and later adapted for local execution.

To run the workflow locally:

  • Ensure that all datasets are available in the /data directory

  • Adjust file paths if necessary

  • Install required Python dependencies

The scripts are intended to be executed sequentially but remain modular.

🔁 Reproducibility statement

This repository provides a complete and transparent implementation of the analytical workflow.

All data transformations, modelling steps, and visualisations are explicitly defined.

The objective is to ensure:

  • reproducibility of results

  • traceability of the analytical process

  • methodological clarity
